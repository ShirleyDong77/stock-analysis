# 📊 股票分析标准工作流

## 触发场景

| 场景 | 触发方式 | 注意事项 |
|------|---------|---------|
| 用户主动要求 | 飞书消息 | 直接执行完整流程 |
| Cron 盘前推送 | 定时任务（9:15 前） | 自动走完盘前 checklist，推送飞书文档 |

## ⚡ 工作流：8步

```
Step 1  → 搜大盘/板块行情（minimax MCP）
Step 2  → 确认核心催化剂
Step 3  → 拉个股实时数据（新浪 API）
Step 4  → 计算技术指标（Python）
Step 5  → 技术面判断
Step 6  → 风险评估（因子评分）
Step 7  → 操作建议
Step 8  → 发飞书文档 + 声明免责
```

## 📁 文档输出规范

**文档命名规则**：
- 盘前分析：`📊 YYYY-MM-DD 股票分析（盘前）`
- 盘后分析：`📊 YYYY-MM-DD 股票分析（盘后）`
- 持仓诊断：`🩺 YYYY-MM-DD 持仓诊断`

**四大输出板块**（Step 8 格式）：
- **一、大盘概况** — 指数表（现价/昨收/涨跌幅/最高/最低）
- **二、催化剂+风险评估** — 传导链条 + 风险评分表 + 推荐方向（受益/规避）
- **三、持仓诊断** — 合并数据表+技术判断+深度分析（每个持仓独立段落，说明为什么）
- **四、情景预测+操作计划** — 关键价位 + 三种情景 + 操作优先级表格 + 盘中时间窗口

## 🔧 Step 1 — 搜索大盘和板块行情

**搜索顺序**（依次尝试，断了就跳到下一个）：
1. **minimax-web-search skill** — 中文搜索效果好 ✅
   ```bash
   python3 ~/.openclaw/workspace/skills/minimax-web-search/scripts/web_search.py "搜索关键词"
   ```
2. `mcporter call minimax.web_search query="关键词"` — 备用，超时30s 下一代
3. 浏览器百度搜索 — 稳定可用
4. `web_fetch` 抓新浪财经/东方财富 — 最后保底

**若全部失败**：在文档中注明"实时搜索失败，以已知催化剂分析"，不卡住流程。

**搜索词（每轮 3-5 个关键词，含日期）**：
```
# 轮次1：大盘整体
"YYYY-MM-DD A股大盘 行情"

# 轮次2：板块逻辑
"中东局势 原油 股价 YYYY年MM月"

# 轮次3：持仓相关（根据 holdings.json 动态调整）
"稀土板块 北方稀土 YYYY年MM月"
"卫星导航 中国卫星 YYYY年MM月"
```

**最少搜 3 轮**，交叉验证。

## 🔧 Step 2 — 确认核心催化剂

从搜索结果中提炼，按重要度排序：

1. **地缘政治** — 战争/制裁/外交事件
2. **美联储/货币政策** — 降息/加息预期、美元走势
3. **国内政策** — 央行/证监会/部委重要表态
4. **行业逻辑** — 产业政策、业绩兑现、供需变化

每个催化剂说明：**传导路径**（这件事 → 怎么影响市场）。

## 🔧 Step 4 — 计算技术指标（Python）

用 Python 一次性计算 6 个量化指标：

```python
# 6个技术指标
收盘位置 = (收盘 - 最低) / (最高 - 最低) × 100%
上影线   = (最高 - max(开, 收)) / (最高 - 最低) × 100%
量比     = 今日成交量 / N日均量
反弹/跌幅比 = |今日涨幅| / |昨日跌幅|
距高点回撤 = (N日高点 - 收盘) / N日高点 × 100%
均线关系  = 收盘 vs MA5 / MA10 / MA20

# 承压判定
强势 = 收盘位置 > 80% + 上影 < 10% + 量比 > 1.0
承压 = 收盘位置 < 60% 或 上影 > 30%
极弱 = 低于 MA5/10/20 + 缩量 + 反弹/跌幅比 < 0.3
```

**预警状态判定**：
- 现价 < 预警下限 → ⚠️ 跌破下轨
- 现价 > 预警上限 → ⚠️ 突破上轨
- 现价 < 下限 × 1.02 → 🔅 接近下轨

## 🔧 Step 5 — 技术面判断

每个持仓股票给出：
1. 今日表现描述（用数据说话，不空口定性）
2. 均线关系（多头排列 / 空头排列 / 纠缠）
3. 支撑位 / 压力位
4. 综合判断：强势 / 承压 / 极弱

## ⚠️ 核心禁忌

1. **不跳步**：8 步必须走完，哪怕时间紧张也要完整
2. **持仓数据不泄露**：holdings.json 内容不写入 SKILL.md，Step 3 代码不硬编码
3. **不空口定性**："看起来很强" → "收盘位置90%，上影线10%，量比1.34x"
4. **不省免责**：每次必须声明"不构成投资建议"
5. **不补仓**在大盘未企稳时：关键支撑位跌破不补仓
